## Why

完成價格追蹤應用程式的前端實作，確保用戶能夠透過網頁介面進行操作。

## What Changes

- 設計並實作 HTML 表單，包含商品名稱、價格與日期欄位。
- 使用 JavaScript 實現表單提交邏輯，並呼叫後端 API。
- 實作價格紀錄列表的渲染邏輯。
- 撰寫前端驗證邏輯，避免無效輸入。

## Capabilities

### New Capabilities

- `frontend-ui`: 用於管理價格數據的前端介面。

### Modified Capabilities

- 無。

## Impact

- 影響 `public/index.html` 和 `public/javascripts/` 檔案。
- 需要新增前端驗證邏輯。
- 需要與後端 API 整合。
